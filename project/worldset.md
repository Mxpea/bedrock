## 世界观设定库（Wiki 模式）全新设计稿 V2.0

### 一、 核心理念：重检索，轻层级
放弃传统的“文件夹-文档”树状结构，改用**“扁平化词条 + 多维标签 + 元数据属性”**的 Wiki 模式。
核心目标是让海量设定能够通过**交叉检索、图谱关联、快速筛选**被瞬间找到，而不是在深深的文件夹里迷失。

---

### 二、 核心数据模型

#### 1. 实体词条表 (`worldview_entries`)
不再是单纯的“文档”，而是结构化的“实体词条”。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | UUID | 主键 |
| `workspace_id` | UUID | 所属工作区 |
| `name` | VARCHAR(100) | 词条名称（如：“永夜城”、“精灵族”） |
| `aliases` | JSON | 别名/曾用名（如：`["The Dark City", "夜都"]`，提高检索命中率） |
| `category` | VARCHAR(50) | 基础分类（如：人物、地点、物品、势>力、设定/规则） |
| `properties` | JSON | 自定义元数据键值对（如：`{"population": "100万", "leader": "夜王"}`） |
| `content` | LONGTEXT | Markdown 正文描述 |
| `created_at` | TIMESTAMP | |
| `updated_at` | TIMESTAMP | |

#### 2. 标签索引表 (`worldview_tags` & `entry_tags_relation`)
用于交叉筛选（如：筛选同时具备 `#阵营-邪恶` 和 `#地理-极寒` 的词条）。

#### 3. 关系网络表 (`worldview_links`)
自动解析正文中的 `[[词条名]]` 语法，建立双向链接。
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `source_id` | UUID | 来源词条 |
| `target_id` | UUID | 目标词条 |
| `context` | VARCHAR(255) | 引用上下文摘要 |

### 三、 交互界面设计：检索与整理中心

#### 3.1 页面布局（控制台 + 预览编辑视图）

```text
+------------------------+--------------------------------------------------+
|      全局过滤中心      |                  词条详情 / 编辑器               |
|      (宽 300px)        |                                                  |
+------------------------+--------------------------------------------------+
| [ 🔍 聚合搜索框...   ] |  【 类别：📍地点 】                             |
| (支持拼音、别名、全文) |                                                  |
|                        |  # 永夜城  (别名: 夜都)                          |
| ⚡ 快速视图 (分类)     |                                                  |
| ---------------------- |  [元数据面板]                                    |
| 🧍 人物 (42)           |  - 从属势力 : {链接至: 暗影议会}                 |
| 📍 地点 (15)           |  - 人口     : 100万                              |

| 📖 规则 (3)            |  + 添加属性...                                   |

| 🏷️ 标签过滤器         |  [正文描述]                                      |

| [x] #极寒地带          |  [[精灵族]] 曾在这里建立过前哨站...              |
```

#### 3.2 核心检索路径

1. **同义词与别名融合检索**：用户搜索“夜都”，搜索框会直接展示“永夜城（夜都）”。
2. **多维交叉筛选**：点击左侧的 `📍地点`，然后再点击标签 `#极寒地带`，列表立刻过滤出符合条件的词条。
3. **网状漫游（双向链接）**：查看“永夜城”时，底部显示所有提及过“永夜城”的其他词条（反向链接），便于梳理复杂的剧情网。

### 四、 编辑体验升级：结构化与非结构化结合

#### 4.1 Notion 式元数据表头 (Properties)
不仅仅是大段文本，用户可以为每个词条添加具象的属性（Properties）。
- 这些属性存储在 `properties` JSON 字段中。
- **作用**：基于这些属性，未来可以生成“人物一览表”、“物品属性对比表格”。

#### 4.2 正文内的智能补全
- 输入 `[[` 弹出词条联想（支持首字母、拼音检索），选中后生成内部链接。
- 如果输入的词条不存在，支持“一键创建占位词条”（Stub），保持思路不中断。

---

### 五、 V2.0 开发重构步骤

1. **数据库重构**：废弃 `worldview_folders`，建立 `worldview_entries`（带 aliases 和 properties）、标签表、链接表。
2. **后端检索 API**：开发聚合搜索接口，支持别名、分类、标签的复合查询。
3. **前端视图层重写**：移除树状组件，引入“多维度侧边栏列表”和“Notion 风格属性表头”。
4. **Markdown 解析器升级**：增加对 `[[词条名]]` 语法的正则表达式拦截，并将其转化为带有 hover 效果的链接。

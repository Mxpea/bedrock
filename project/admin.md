管理员后台是整个平台的**控制中枢**。设计目标是：**高效管控、安全隔离、数据透明、最小侵入**——管理员能快速处理待办事项、监控异常，但对作者端的创作体验不产生额外干扰。

以下从信息架构、核心页面布局、操作流程三个维度进行设计。

---

### 一、管理员后台信息架构（侧边栏导航）

管理员登录后，顶栏多出一个控制台按钮。

控制台界面：

```
[平台 Logo]
══════════════════
📊 仪表盘
📁 工作区管理
👥 用户管理
🛡️ 审核队列
   ├── 高级权限申请
   ├── 公开内容审核（可选）
   └── 举报处理 
💬 吐槽监管
⚙️ 系统设置
   ├── 注册策略
   ├── 邀请码管理
   ├── 安全策略
   └── 全局样式（可选）
📈 数据看板
🔧 运维工具（可选）
══════════════════
[管理员头像 / 退出]
```

### 二、核心页面详细设计

#### 1. 仪表盘（Dashboard）

**定位**：登录后首页，呈现待处理事项与关键指标。

**布局**：卡片式网格 + 快捷操作入口。

| 区域 | 内容 |
| :--- | :--- |
| **待办提醒** | 高级权限申请待审数、举报待处理数、异常登录告警（如有）。 |
| **关键指标卡片** | 今日新增用户、今日新增工作区、今日公开章节数、今日吐槽数。 |
| **实时动态流** | 显示最新创建的工作区、最新公开的作品、最新高级权限申请。 |
| **快捷入口** | 「生成邀请码」「切换注册模式」「审核队列」。 |

**交互**：点击卡片数字直接跳转对应列表页。

---

#### 2. 工作区管理

**功能**：查看、搜索、筛选、编辑、锁定全站所有工作区（无视作者隐私锁）。

**列表字段**：
- 工作区封面/名称
- 所属作者（点击跳转该用户详情）
- 可见状态（私密/内部预览/公开）
- 章节数/字数
- 最后更新时间
- 操作按钮：**查看详情**、**强制设为私密**、**锁定（禁止编辑）**

**搜索/筛选**：
- 按工作区名称、作者名/ID 搜索。
- 按可见状态筛选。
- 按创建时间范围筛选。

**详情页（点击工作区进入）**：
- 可查看该工作区下的**正文章节列表、大纲结构、人物卡片、世界观条目**（只读模式，但管理员可强制编辑）。
- 提供「强制下线预览链接」「清除所有缓存」「导出为 EPUB」等高级操作。

---

#### 3. 用户管理

**列表字段**：
- 用户头像/昵称/ID
- 注册时间/最后登录 IP
- 角色（读者 / 作者 / 编辑 / 管理员）
- 高级自定义权限等级（Lv.1/Lv.2/Lv.3）
- 账号状态（正常 / 封禁）
- 操作：**编辑角色**、**调整权限等级**、**封禁/解封**、**查看主页**

**用户详情页**：
- 基本信息 + 账号安全日志（登录记录）。
- 该用户拥有的**工作区列表**（可批量操作）。
- 该用户的**吐槽/评论记录**。
- 该用户的**主页装修配置快照**（管理员可重置为默认）。

**批量操作**：勾选用户后可批量赋予角色、批量封禁。

---

#### 4. 审核队列

**4.1 高级权限申请（Lv.3）**

| 列表字段 | 说明 |
| :--- | :--- |
| 申请人 | 点击跳转用户详情 |
| 申请时间 | 支持排序 |
| 申请理由 | 截断显示，悬浮全显 |
| 测试工作区 | 点击预览该工作区（新标签页打开阅读视图） |
| 操作 | **通过** / **拒绝** / **查看代码** |

**审核详情抽屉**：
- 显示申请理由全文。
- **代码安全扫描结果**（自动检测是否含 `eval`、`document.cookie`、外链等）。
- **沙箱预览区**：渲染作者提交的自定义代码效果。
- **审核意见输入框**（拒绝时必填）。
- **操作记录**：谁在何时通过了该申请。

**4.2 公开内容审核（可选模块，按需开启）**
若系统开启“公开作品需审核”，此处展示待审工作区，管理员可预览后批准公开。

**4.3 举报处理**
读者可举报作品内容、主页模块、留言。管理员在此处理举报，可选择**忽略**或**删除内容并通知作者**。

---

#### 5. 吐槽监管

**定位**：全站划线吐槽内容的总控台，主要用于防止内部审稿出现违规言论。

**列表视图**（类似工单系统）：
- 吐槽内容（截断）
- 所属工作区/章节
- 吐槽人 / 被@人
- 可见范围（内部公开/编辑组/私密）
- 发布时间
- 状态（正常 / 被举报 / 已屏蔽）
- 操作：**查看上下文**、**屏蔽**、**删除**

**筛选**：按可见范围筛选（管理员有权查看包括“私密”在内的所有吐槽）。

**详情抽屉**：
- 吐槽全文及所在章节的上下文。
- 该吐槽下的回复列表（支持折叠/删除）。
- 快速跳转至章节阅读页。

---

#### 6. 系统设置

**6.1 注册策略**
- 开关：「开放注册」/「仅邀请码注册」。
- 切换时弹窗二次确认。
- 注册默认角色选择：读者（默认）/ 作者（需审核）。

**6.2 邀请码管理**
- 生成新批次：设置前缀、可用次数、有效期（永久/指定日期）。
- 列表展示：批次号、生成时间、已用次数/总次数、状态（有效/已用完/已过期）、操作（提前失效、导出未使用码）。
- 支持一键复制单个邀请链接（格式：`https://平台域名/register?invite=XXXX`）。

**6.3 安全策略**
- 高级自定义权限（Lv.3）的审核模式：手动审核 / 自动通过（高风险）。
- 沙箱隔离强度配置（iframe `sandbox` 属性值预设）。
- 敏感词词库管理（增删改查，支持导入导出）。
- IP 黑名单/白名单。

**6.4 全局样式（可选）**
- 平台主色调、Logo、Favicon 替换（适用于私有化部署或品牌定制）。

---

#### 7. 数据看板

**定位**：仅管理员可见的全平台数据分析。

**时间维度切换**：今日 / 本周 / 本月 / 自定义。

| 数据模块 | 指标 |
| :--- | :--- |
| **用户增长** | 新增注册数、活跃作者数（有过编辑行为）、DAU/MAU 趋势图。 |
| **内容生产** | 新建工作区数、新增章节数、总字数增量、人均章节数。 |
| **阅读消费** | 公开章节 PV/UV、平均阅读时长、跳出率（仅限公开作品）。 |
| **协作活跃度** | 划线吐槽总数、人均吐槽数、吐槽回复率。 |
| **异常监控** | 失败登录次数、被举报内容数、敏感词命中次数。 |

**图表类型**：折线图、柱状图、饼图（用 ECharts 或类似库）。

**导出功能**：所有图表数据可导出为 CSV 或 PNG。

---

#### 8. 运维工具（折叠菜单，高级管理员可见）

- **缓存清理**：一键清除全站静态资源缓存、工作区渲染缓存。
- **文档转换服务状态**：显示 Gotenberg/LibreOffice 服务健康状态，支持重启。
- **系统日志**：最近 500 条管理员操作日志（谁在何时做了什么操作）。
- **强制重新索引**：用于搜索功能数据重建。

### 三、管理员操作权限矩阵

| 操作 | 超级管理员 | 普通管理员（可配置角色） | 编辑 |
| :--- | :--- | :--- | :--- |
| 查看数据看板 | ✅ | ✅ | ❌ |
| 管理用户（封禁/改角色） | ✅ | ✅（不可修改超管） | ❌ |
| 审核高级权限 | ✅ | ✅ | ❌ |
| 查看/屏蔽所有吐槽 | ✅ | ✅ | ✅（仅限所负责工作区） |
| 修改系统设置 | ✅ | ❌ | ❌ |
| 查看私密工作区内容 | ✅ | ✅（需二次授权） | ✅（仅限协作区） |

### 四、交互设计原则

- **所有高危操作**（封禁、删除、强制私密）必须有二次确认弹窗。
- **管理员查看私密内容时**，页面顶部显示醒目横幅：「你正在以管理员身份查看私密内容，请注意隐私保护」。
- **批量操作**支持勾选多行，并提供「全选当前页」「反选」。
- **列表页支持列自定义显示**（勾选想看的字段），并记住用户偏好。
- **暗色模式支持**：管理员后台长时间使用，建议提供暗色主题切换。

---

### 五、与前端规划的衔接

- 管理员后台为**独立前端路由**（如 `/admin`），与作者端、阅读端完全分离，不加载作者端装修样式。
- 后台 UI 风格建议采用**企业级中后台设计规范**（如 Ant Design Pro 风格），强调信息密度和操作效率，而非个性展示。

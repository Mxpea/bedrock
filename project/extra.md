### 模块细化九：分级自定义权限体系（安全沙箱与解锁通道）

我们将作者分为两个层级：**标准创作者** 与 **认证高阶作者**。

#### 层级一：标准创作者（默认权限）

- **可用范围**：只能使用我们在编辑器里内置的 **Markdown 扩展语法按钮** 和 **CSS 变量选择器**。
- **Markdown 扩展语法（内置安全库）**：
    - `{高亮|文字}` -> 黄色背景。
    - `{波浪|文字}` -> 波浪下划线。
    - `{模糊|文字}` -> 高斯模糊（悬停才清晰，用于剧透预警）。
    - `{注音|汉字|拼音}` -> 上方显示拼音（轻小说/少儿模式）。
    - `{刮刮卡|文字}` -> 黑条遮盖，鼠标悬停显示（悬疑互动）。
- **CSS 白名单预设**：在“页面装修”界面，只能拖动滑块调整**我们预设的变量**：
    - `--page-bg-color`
    - `--text-font-family` (从我们内置的 10 种安全字体选)
    - `--link-color`
    - `--paragraph-spacing`

#### 层级二：认证高阶作者（需申请解锁）

**申请与审核流程：**
1. 作者后台点击 **“申请高级样式权限”**。
2. 填写理由（例如：“我想实现古籍竖排效果”、“我想为科幻作品定制动态星空背景”）。
3. **上传一个 `.txt` 文件**，内含其计划使用的**自写 CSS 代码段**。
4. 管理员在后台查看申请，**进行人工代码审计**（重点排查 `url()` 外链、`expression`、`@import`、大范围绝对定位）。
5. 审核通过后，为该作者的**账号**或**特定作品**开启 **“自定义 CSS 注入”** 开关。

**解锁后的能力与约束：**
- **能力**：可编写任意 CSS 代码（后端仍会运行**自动过滤防火墙**，拦截已知危险函数，但允许 `transform`、`animation`、`grid` 等高级布局）。
- **风险告知**：解锁时需签署电子确认书：“若因自写 CSS 导致页面排版错乱、内容遮挡，后果自负；若发现恶意代码，永久封禁账号。”
- **监控机制**：系统后台自动扫描已解锁作品的 CSS。若检测到 `position: fixed; top: 0; left: 0; width: 100vw; height: 100vh` 这种**覆盖全屏劫持点击**的代码，系统发出告警并自动**临时回滚**为标准样式。

#### 自定义 Markdown 语法扩展（高阶）

对于解锁用户，允许在编辑器内使用**有限的 HTML 标签**（基于 DOMPurify 白名单）：

- **允许的标签**：`<ruby>`, `<span>`, `<div>`, `<i>`, `<b>`, `<br>`, `<hr>`。
- **允许的属性**：`class`, `id`, `title`。
- **禁止行为**：`<script>`, `onclick`, `style` 属性（样式必须写在前文提到的独立 CSS 框内）。

### 模块细化十：作者主页的“完全自定义”延伸

基于分级体系，**主页自定义**同样适用：
- **标准作者**：只能更换头图、头像，从 6 种布局模板中选择。
- **高阶作者**：解锁后，可上传自定义 **HTML/CSS 单页**作为主页（例如像 Notion 个人页那样），但这部分内容将渲染在 **沙箱 iframe** 中（`sandbox="allow-scripts allow-same-origin"`），且与主站数据交互受严格限制（仅能通过 `postMessage` 获取作品列表 JSON）。

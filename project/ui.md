从产品规划转入交互与视觉设计，核心原则是：**为内部作者提供沉浸式创作感，为读者提供纯净阅读感，为管理员提供高效管控感**。以下按角色和场景构建交互蓝图。

---

### 一、全局导航架构（信息架构 IA）

不同角色登录后看到的**主导航菜单完全不同**，这是设计第一准则。

| 角色 | 顶部导航栏菜单项 | 侧边栏/个人面板入口 |
| :--- | :--- | :--- |
| **未登录访客** | 首页、发现（公开作品库）、关于 | 登录 / 注册 |
| **读者（普通用户）** | 首页、发现、我的书架 | 个人中心、阅读历史 |
| **作者（内部创作者）** | **创作工坊**、我的作品、自有数据库、消息 | 头像下拉：主页装修、权限申请、账号设置 |
| **编辑/管理员** | **管理面板**、全站作品、用户管理、审核队列 | 头像下拉：系统配置、数据看板 |

**首页设计策略**：
- 未登录：展示**精选公开作品**（大封面流）+ 平台气质文案。
- 作者登录后：首页直接变成**工作台视图**（最近编辑作品、待处理吐槽、创作日历、快捷入口）。

### 二、核心页面交互设计与入口设置

#### 1. 创作工坊（作者核心操作中枢）

这是作者登录后的默认落地页，设计目标是**一站式进入写作流**。

**布局建议**：三栏式（左导航 / 中内容 / 右预览）

- **左侧竖栏（模块切换）**：
  - 图标导航：📝 编辑器、📚 作品管理、🗂️ 自有数据库、🎨 作品装修、⚙️ 作品设置。
  - 悬浮态说明文字。

- **中间区域（最近动态与快速入口）**：
  - 大号按钮：**「+ 新建作品」**（点击弹出：填写书名、简介、选择封面）。
  - 卡片列表：「最近编辑的章节」，每个卡片显示：作品名、章节名、最后编辑时间、**继续写作**按钮。
  - 吐槽通知条：显示「有 3 条新吐槽待处理」，点击跳转吐槽汇总。

- **右侧面板（写作辅助）**：
  - **迷你数据库**：显示最近编辑过的人物/设定书签，可快速查看摘要。
  - **写作统计（仅字数）**：今日字数、本周字数（无阅读数据）。

**交互细节**：
- 点击「继续写作」直接进入**分屏编辑器**（左侧 Markdown 源码，右侧实时渲染预览）。
- 编辑器顶部工具栏**常驻**，提供：加粗、斜体、标题、引用、插入图片、**「插入数据库条目」**（弹出搜索框选择人物/设定）。

---

#### 2. 阅读端交互（含划线吐槽）

**页面结构**：
- **顶部沉浸栏**：半透明悬浮，显示章节标题、作品名。鼠标悬停出现**目录按钮**和**设置按钮**。
- **正文区域**：居中、最大宽度 700px 左右，背景色由作者自定义。
- **右侧悬浮操作条**（距离正文右侧 20px）：
  - 📝 划线模式开关（开启后选中文字自动弹出批注框）
  - 💬 查看本章批注（展开右侧边栏显示该章所有划线吐槽）
  - 🎵 音效开关（若作者设置了 BGM）
  - 🔖 书签（收藏本章）

**划线吐槽交互流程**：
1. 读者选中一段文字，浮现小菜单：「写想法」、「划线」、「分享」。
2. 点击「写想法」，右侧滑出半屏面板，显示选中文字引用，下方输入框。
3. 发布前可选择可见范围（仅内部/编辑组/私密），点击发送。
4. 被批注的句子出现浅黄色背景高亮，鼠标悬停显示吐槽者头像和内容预览。

**访客/无权限用户**：选中文字无任何反应。

---

#### 3. 自有数据库（大纲/设定/人物）

**入口位置**：
- 顶部导航「自有数据库」或编辑器内右侧「📚」图标。

**页面布局**：
- 顶部 Tab 切换：**大纲** | **人物** | **设定** | **书签**。
- 左侧树状目录（针对大纲）：可拖拽节点调整顺序，右键菜单重命名、删除、添加子节点。
- 右侧详情编辑区：Markdown 编辑器，支持上传图片。

**搜索与书签交互**：
- 全局搜索框位于右上角，输入关键词实时过滤三大模块结果，结果分栏显示。
- 每个条目卡片右上角有「☆」按钮，点亮后加入「书签」视图。

**与编辑器联动**：
- 在编辑器内输入 `@` 触发人物/设定搜索下拉菜单，选择后插入一个标签（如 `[角色:林雪]`），渲染为可点击的小胶囊，点击跳转数据库对应条目。

---

#### 4. 作品装修与主页自定义（可视化搭建器）

**入口**：作品管理 -> 点击作品卡上的「🎨 装修」按钮。

**界面设计**：仿照低代码平台，分为**预览画布**和**右侧属性面板**。

- **画布顶部**：切换预览模式（PC/手机/平板）。
- **左侧组件库**：背景设置、字体样式、布局模板、音效上传、自定义 CSS（白名单校验）。
- **右侧属性面板**：当前选中元素的参数调节（颜色取色器、滑块、开关）。

**高级用户解锁自定义**：
- 若作者拥有 Lv.3 权限，属性面板多一个「高级」Tab，可输入原始代码，并有「在沙箱中测试」按钮。

**保存与发布**：
- 装修完成后点击「应用到作品」，实时生效于该作品所有章节。

### 三、UX 设计原则（风格指南）

| 维度 | 设计策略 |
| :--- | :--- |
| **色彩** | 主色建议**低饱和度中性色**（灰、白、米色），避免视觉疲劳。强调色使用深蓝或墨绿，代表专业与沉静。**拒绝亮红、亮橙**（焦虑色）。 |
| **字体** | 正文默认**思源宋体** / **霞鹜文楷**（有书卷气）。编辑器使用等宽字体 **JetBrains Mono** 或 **Fira Code**。 |
| **动效** | 所有过渡使用 `ease-out` 缓动，**禁用弹跳、旋转等花哨效果**。页面切换采用**淡入淡出**，保持心流。 |
| **响应式** | PC 端为第一优先级（作者主要使用电脑写作），移动端保证**阅读体验**（字体大小自适应），编辑器不要求移动端可用。 |
| **空状态设计** | 当作者没有作品或数据库为空时，显示**温和的插画 + 引导按钮**，而非空白页面。 |

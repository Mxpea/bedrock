这是一个关键的架构升级。将原先松散的“作品+数据库”整合为**以项目为中心的封闭工作区**，能极大降低作者在多个作品间切换时的认知负担，数据隔离也更清晰。

以下是对交互设计、入口设置及页面结构的重构方案。

---

### 一、核心概念转变：从“作品”到“工作区”

| 旧概念 | 新概念（工作区 Workspace） |
| :--- | :--- |
| 一部小说 = 一个作品 | **一个项目 = 一个工作区** |
| 大纲/人物/设定是全局共享库 | 大纲/人物/设定**仅属于当前工作区**（数据完全隔离） |
| 编辑正文需退出到列表 | 在工作区内**无缝切换模块** |

**作者视角的一句话理解**：
> “我创建一个工作区来写《三体》，这个工作区里有它的正文、大纲、人物、世界观。我再创建一个工作区写《流浪地球》，两者互不干扰。”

----

### 二、全局导航与入口调整

#### 1. 顶部主导航（作者登录后）

| 菜单项 | 功能说明 |
| :--- | :--- |
| **🏠 工作台** | 展示所有工作区卡片列表，新建工作区入口。 |
| **📬 消息** | 吐槽通知、系统通知。 |
| **👤 个人** | 主页装修、权限申请、账号设置。 |

> **设计要点**：主导航不再出现“自有数据库”，因为数据库已下沉到每个工作区内部。

#### 2. 工作区切换器（全局组件）

在进入任一工作区后，**顶部栏中央**显示当前工作区名称，带下拉箭头。

- 点击下拉箭头，浮层展示：
  - 「我的所有工作区」列表（最多显示 5 个，带搜索框）。
  - 「+ 新建工作区」按钮。
- 切换工作区时，**记忆用户上次在该工作区停留的页面**（例如上次编辑到第三章，切换回来直接打开第三章编辑页）。

---

### 三、工作区内部布局：沉浸式项目管理界面

进入某个工作区后，页面采用**左侧竖栏导航 + 右侧内容区**的经典结构。

#### 1. 左侧导航栏（工作区专属菜单）

```
[工作区名称]
══════════════════
📖 正文
   ├── 章节列表
   └── 新建章节
🗺️ 大纲
👥 人物
🌍 世界观
══════════════════
🎨 外观定制
⚙️ 工作区设置
══════════════════
← 返回工作台
```

- 点击任意模块（正文/大纲/人物/世界观），右侧内容区**独立全屏展示该模块的编辑界面**。
- 当前所在模块高亮显示。

#### 2. 面包屑导航（辅助定位）

右侧内容区顶部显示面包屑，例如：
> 工作区：三体 / **正文** / 第二章·射手与农场主

这能让作者清晰感知当前位置，避免在深层页面中迷失。

---

### 四、各模块独立编辑页详细设计

#### 1. 正文模块编辑页（双栏结构）

这是最高频使用的页面。

- **左侧**：章节树状列表（可拖拽排序、右键菜单重命名/删除/新建子章节）。
- **中间**：Markdown 编辑器（全屏沉浸式，支持分屏预览切换）。
- **右侧**：可收缩的侧边栏，包含三个 Tab：
    - **章节信息**（字数、创建时间、状态）。
    - **本章吐槽**（显示该章所有划线批注，作者可在此直接回复）。
    - **快速引用**（可搜索本工作区的人物/设定，拖入编辑器）。

> **入口路径**：点击左侧导航“正文” → 中间章节列表选中某一章 → 右侧加载编辑器。

#### 2. 大纲模块编辑页（思维导图式或树状大纲）

- **视图切换**：顶部提供「树形大纲」和「思维导图」两种模式。
- **树形大纲模式**：类似 Workflowy 或 Dynalist，无限层级缩进，每行支持 Markdown 简写，拖拽调整节点。
- **思维导图模式**：基于树形数据自动生成可缩放拖拽的导图，节点可双击编辑。
- **关联章节**：大纲节点可右键「关联已有章节」，建立双向链接（在大纲中点击跳转到对应正文）。

#### 3. 人物模块编辑页（卡片墙 + 详情抽屉）

- **主视图**：**人物卡片墙**。每张卡片显示头像（可上传）、姓名、一句话简介。
- **搜索与过滤**：顶部搜索框，可按姓名、标签筛选。
- **新建人物**：点击「+ 新建人物」卡片位，弹出快速创建表单（姓名必填）。
- **编辑详情**：点击卡片，右侧滑出**详情抽屉**（占屏幕 40% 宽），包含：
    - 基本信息（姓名、别名、性别、年龄、外貌）。
    - 人物关系图谱（可在此页内直接连线其他人物）。
    - 出场章节记录（自动关联正文中的 `@人物` 引用，形成反向索引）。
    - 自由备注区（Markdown 编辑器）。
- **书签**：卡片右上角点亮星标，置顶显示在卡片墙前列。

#### 4. 世界观模块编辑页（结构化 Wiki）

- **主视图**：左侧分类树（如：地理、历史、势力、魔法体系），右侧条目列表。
- **条目编辑**：点击条目进入**独立全屏编辑页**（类似 Notion 页面），支持上传图片、引用人物/大纲。
- **引用面板**：每个条目底部自动显示「被以下章节引用」，方便作者审查设定一致性。

---

### 五、工作区外观定制（独立于各模块）

点击左侧导航「🎨 外观定制」，进入**可视化装修器**（沿用此前设计）。

- 装修效果**作用于该工作区内所有正文阅读页**（即读者最终看到的展示页）。
- 预览画布实时显示当前工作区的任意章节（默认第一章）。
- 保存后，该工作区下的所有章节阅读页统一换装。

---

### 六、工作区设置页

点击左侧导航「⚙️ 工作区设置」，右侧显示：

- **基本信息**：工作区名称、简介、封面图、标签。
- **可见性控制**：
    - 私密（仅自己和管理员可见）
    - 内部预览（生成临时访问链接，可设置有效期和密码）
    - 公开发布（出现在发现页）
- **协作成员**（可选扩展）：可邀请其他作者共同编辑（需对应权限设计，初期可不做）。
- **删除工作区**（高危操作，二次确认）。

---

### 七、调整后的用户流程示例（作者从零开始）

1. 登录 → 工作台 → 点击「新建工作区」→ 填写名称《新世界》→ 创建成功自动进入工作区。
2. 在工作区内，左侧导航点击「👥 人物」→ 新建「主角·陈末」→ 完善详情。
3. 点击「📖 正文」→ 新建章节「第一章·醒来」→ 在编辑器中输入文字，用 `@陈末` 插入人物引用。
4. 写完点击「保存」→ 点击「外观定制」→ 选一个深色背景 → 点击「发布」。
5. 在工作区设置中生成「内部预览链接」，发送给编辑审稿。

### 八、对现有规划文档的调整摘要

| 原概念 | 调整后 |
| :--- | :--- |
| “自有数据库”为全局 | **删除全局数据库**，转为工作区内的“人物+世界观”模块。 |
| 作品装修与正文分离 | 装修入口位于工作区左侧导航，作用于整个工作区的阅读页。 |
| 搜索书签跨作品 | 搜索书签仅限**当前工作区内**，各工作区数据完全隔离。 |
